# SPECIFICATION

## 1. 概要
Three.js (WebGL) を用いた、アンティーク・クラシックな洋書風デザインの3D本オープニングアニメーションと、インタラクティブな本棚（Bookshelf）のフロントエンド実装。
本をクリックすると中央で開き、その3D見開きページの上にHTML/CSSによるコンテンツ（ロゴ、フォーム、掲示板、SNSリンク）が完全に重なって表示される、高度に同期したSPA構成。

## 2. UX仕様（演出フロー）
### 2.1. オープニングシーン
* **初期状態**: 空間中央に閉じたアンティーク調の3Dの本が配置されている。
  * タイトルはゴールド箔押しで **「TEDxUTsukuba Salon \n Library」** と刻印されている。
* **アニメーション**:
  1. **本の手前引き寄せ (第1段階/自動)**: ページロード時、自動的に本がカメラに向かって滑らかに近づいてきて、画面中央の所定の位置で停止する（表紙は閉じたまま）。
  2. **タップ促し演出 (待機状態)**:
     * ズームインが完了すると、画面中央にぼかしを強く効かせた金色の円（CSSの `filter: blur(4px)` を適用し、エッジを完全に溶かして水面に広がるリアルな光の波紋を表現したアニメーション効果、最大スケール10.0）と、「タッチで本を開く」というテキストがフェードインし、ユーザーのタップを待つ。
  3. **ユーザーインタラクションによる開本 (第2段階)**:
     * **タッチ有効判定エリアの制限**: クリック判定を画面全体ではなく、波紋が広がる範囲（画面中央から半径 `150px` の円内エリア）に制限する。この有効エリア外がタッチ/クリックされても反応しない。
     * **動的マウスカーソル制御**: PC環境等でマウス移動時、カーソルが画面中央の有効判定エリア（半径 `150px` 以内）に入った時のみ `pointer`（指マーク）になり、エリア外では `default`（矢印マーク）になるように制御する。
     * **開本と衝撃波のトリガー**: 有効判定エリア内がタッチ/クリックされると、クリックの衝撃波として画面中央から特大の金色のクリック波紋（こちらも `filter: blur(4px)` を適用して滑らかにぼかし、2枚がわずかに遅延して追いかけるように画面いっぱいに広がるアニメーション）がブワッと発生し、タップ促しプロンプトとテキストが消える。同時に本が開くアニメーションが開始され、表紙が左側を軸に180度滑らかに開ききる。
  4. **ページダイブとホワイトアウト**: カメラが本の内側（ページ）に向かって急接近し、ページの中に飛び込むようなズームイン効果が起き、画面全体がホワイトアウトする。
  5. **ホワイトアウトの維持**: 完全に真っ白になった状態を0.8秒間維持し、余韻を持たせる。
  6. **本棚シーンへの移行とズームインカメラワーク**:
     * ホワイトアウト中に裏で本棚オブジェクトを構築し、カメラを目標本棚アングルより「少し遠方かつ高い位置」（Z軸に+2.5、Y軸に+0.3）に配置する。
     * ホワイトアウトからのフェードアウト（1.5秒）と同調して、カメラが本棚の中心（0, 0, 0）を注視したまま目標アングルまで滑らかにズームイン（2.2秒、ease: "power2.out"）する。フェードアウトが完全に終了した後も、わずかにカメラの動きの余韻を残してピタッと静止する演出とする。

### 2.2. メインシーン（本棚）
* **本棚構造**: 木製の本棚に合計8冊の洋書が背表紙を向けて隙間なく格納されている。
  * **スロット構成 (配置順)**:
    1. ダミー本1 (Archive Vol.1 / 緑 / ダミー)
    2. ドキュメント本2 (Docs Vol.2 / 紫 / ダミー)
    3. パートナー本 (Our Partners / 赤 / **アクティブ**)
    4. アンケート本 (Questionnaire / 紺 / **アクティブ**)
    5. コメント本 (Comments / 深緑 / **アクティブ**)
    6. アカウント本 (Social Accounts / 金 / **アクティブ**)
    7. ギャラリー本3 (Portfolio / 青 / ダミー)
    8. エッセイ本4 (Essays / 茶 / ダミー)
  * **ダミー本のインタラクション無効化**:
    * 左右の合計4冊のダミー本は、表紙・背表紙ともにタイトル等の文字テキストは一切描画されず、無地（外枠の金箔装飾のみ）の状態で配置される。
    * マウスホバー時の浮き上がり（growアニメーション）やクリックによる展開が一切無効化されている（マウスカーソルも `pointer` にならない）。ただし、アクティブ本が開いた際には本棚の他の本と同様に奥に引いてフェードアウトする。
  * カメラは少し引いた位置から本棚全体を見渡す。
  * 背表紙の文字は正対縦積みレイアウト（大型化された `76px` フォント、黒フチ、影）で高い可読性を維持。
  * **本棚シーンのUI非表示**: 画面のノイズを低減し没入感を高めるため、ヘッダー・フッター等のUI文字テキスト（「ARCHIVE」、「Select a book to explore」、「Antique Portfolio」等）はすべて削除。


### 2.3. インタラクション (クリック展開と閉じる挙動)
* **クリック展開（3D本のページとHTMLコンテンツの完全同期）**:
  1. **本棚からの引き抜きフェーズ (隣書とのめり込み防止)**:
     * 本棚から本が引き出される際、隣の本のメッシュと交差するのを防ぐため、まずはX座標、Y座標、および回転（背表紙が手前を向いたY軸90度の状態）を完全に維持したまま、Z軸のみを大きく手前に前進（Z=0からZ=1.5へスライド）させて、本棚の束から完全に引き抜く。
  2. **中央配置＆回転フェーズ**:
     * 完全に引き抜かれた後、本を画面中央（X=0, Y=0, Z=targetZ）へスライドさせながら、同時にY軸の回転をMath.PI/2から0へと変化させ、表紙を正面に向ける。
  3. **表紙全開**:
  3. **本が開くダイナミックな接近演出 (ページがカメラに迫るズーム)**:
    * 表紙が開ききる動作と同調して、カメラが最終ズーム距離 (`cameraTargetZ`) まで急接近する。これにより、3Dの本の見開きページが画面いっぱいに迫ってくるようなダイナミックなページダイブ感を生み出す。
  4. **全面真っ白な HTML オーバーレイのシームレスな融合**:
    * 透過表示や本風のテクスチャ（折り目や境界線、陰影など）はすべて廃止し、画面全体を覆う純白のフルスクリーン背景（`background: #ffffff`）を使用する。
    * 本が開きカメラが急接近するアニメーションの終盤（開ききる0.6秒前）に、HTMLオーバーレイを滑らかにフェードインさせ、3Dの本の白い面が画面全体に拡大してフラットな白背景にシームレスに同化するトランジションを表現する。
    * デスクトップ環境では、左右の2カラム（`.page-side`）が中央に美しく整列され、最大幅 `1000px` の中に読みやすく配置される。
    * スマートフォン（縦長画面）では、横並びを解除して1カラムの縦積み構成（`flex-direction: column`）にし、コンテナ全体をスクロール可能にして圧倒的な可読性を確保する。
  5. 画面右上にアンティークゴールド調の「戻る（✕）」ボタンがフェードインする。
* **戻る（閉じる）挙動**:
  1. 「戻る」ボタンをクリックすると、オーバーレイとボタンがフェードアウト。
  2. **表紙のクローズ**: 本の表紙が0.8秒かけて完全に閉じる。
  3. **元のX座標位置への退避＆回転**:
     * 本を閉じた状態のまま、元の本のXスロットの真上（Z=1.5の位置）へ移動させつつ、同時にY軸回転を90度に戻して背表紙を手前に向ける。この時点ではまだ本棚から飛び出た状態。
  4. **押し込みフェーズ**:
     * スロットの上空（Z=1.5）から、本棚の奥（Z=0）へ向かって真っ直ぐZ軸上をスライドさせて元のスロットへ押し込む。これにより、隣の本と物理的に干渉せずに元通りに収納される。
  5. **不透明度の復元**: 退避していた他書とカメラの位置も本棚初期アングルに復帰。
  6. **光のクリーンアップ (ライトリーク防止)**: 開本時に作成した PointLight (`bookGlow`) の強度 (`intensity`) を、閉本アニメーション中に `0` にフェードアウトさせ、完了時にメッシュから削除・`dispose()` を行い、本棚シーンのライティング累積バグ（開閉するたびに空間全体が明るくなる問題）を防止する。
  7. **二重入力とイベント貫通の防止 (3D破損バグ対策)**:
    * 戻るボタン押下時、即座に `isTransitioning = true` を設定してアニメーション完了まで再入力を完全にブロック。
    * `#content-overlay` および `#close-button` で `pointerdown` 時のイベント伝播を停止 (`stopPropagation()`) させ、HTML UI クリックが背面の 3D 空間レイキャスターに貫通して本が誤検知・誤動作（変形バグ）を起こす問題を完全に防止する。

## 3. 技術仕様
* **言語/ライブラリ**: HTML5, CSS3, JavaScript, Three.js (CDN), GSAP (CDN)
* **HTMLコンテンツオーバーレイ (`#content-overlay`) の全面フラット白背景**:
  * 全面フラット白背景 (`background: #ffffff`)、余計な境界線や折り目シャドウは全廃。
  * デスクトップ環境: flex-directionは横並び (`row`)、左右のページ幅は `50%` (max-width: 500px)、高さ `80%` で中央揃え。
  * モバイル環境: flex-directionは縦積み (`column`)、幅 `100%`、高さ `auto`、全体スクロール (`overflow-y: auto`) を有効にし、文字サイズと余白を大きく確保。
  * このHTMLサイズに対し、3Dの本の見かけの表示サイズを合わせるため、カメラZ距離（`cameraTargetZ`）と本Z位置（`targetZ`）を以下のように最適化:
    * 横画面 (PC): 本Z=`2.2`, カメラZ=`3.8` (見かけの距離 `1.6`)
    * 縦画面 (スマホ): 本Z=`1.4`, カメラZ=`3.9` (見かけの距離 `2.5`)
* **歪みのない Canvas テクスチャ生成 (アスペクト比同期)**:
  * 3Dモデル表面の引き伸ばし（縦横比の不一致）による文字・輪郭線の歪みを防ぐため、テクスチャCanvasのサイズを最適化する:
    * **背表紙 (Spine)**: `160 x 1024` (アスペクト比 `1 : 6.4`、メッシュ寸法 `0.5 x 3.2` に対応させ歪みを完全排除)
    * **表紙 (Cover)**: `1024 x 1024` (元の余裕のあるアスペクト比を維持し、長いタイトル文字列「TEDxUTsukuba Salon Library」が枠線と重ならず綺麗に収まるレイアウトスペースを確保)
* **背表紙の装飾 & 縦積み Canvas テキスト**:
  * 金色の帯および細線枠の余白を `160px` 幅に合わせて最適化（帯余白 `25px`、枠線余白 `30px`、枠線幅 `3px`）。
  * 8文字以下の基本サイズ `76px` / `lineHeight 86px`。
  * 黒フチの線幅を `lineWidth = 4` に縮小し、シャドウも `shadowBlur = 4` / `shadowOffset = 1.5` へと大幅に抑えて上品かつ極めてクッキリとした可読性を維持。
* **コード構造**: 単一のファイル（`index.html`）に内包。

### 2.4. 外部リンクへのリダイレクト仕様
* **外部URL設定**: HTML上部の定数定義エリア（`EXTERNAL_LINKS`）にて、`survey`（アンケート）および `comments`（コメント）の遷移先URLを簡単に変更可能。
* **ポップアップブロック対策**: ユーザーが本棚のアクティブ本（Survey, Comment Screen）をダブルクリックした瞬間（ユーザー操作イベントスレッド内）に `window.open(url, '_blank')` を呼び出し、ブラウザのポップアップ遮断機能を確実に回避する。
* **画面ダイブと案内表示**:
  * 外部リンクを開く際も、元のタブでは3Dの本が開きカメラが急接近する美しいオープニングトランジションが通常通り再生される。
  * 本が完全に開いた白い画面（`#content-overlay`）には、「別タブで開いています」というステータス案内、手動で再試行するための「アンケートを開く / Comment Screenを開く」ボタン、および本棚に戻る「戻る（✕）」ボタンが表示される。
  * 外部URLが設定されていない（空文字 `""`）場合は、従来のインラインアンケートフォームやコメントボード（掲示板）がそのまま表示される親切なフォールバック設計。

### 3.1. 背表紙の複数列（改行）描画仕様
* **背表紙タイトル改行対応**:
  * 背表紙タイトル（`spineTitle`）に改行コード `\n` または `\\n` が含まれている場合、キャンバス内で自動的に複数列（サイド・バイ・サイド）の縦積みテキストとして分割・配置される。
  * **レイアウト調整**:
    * 2列配置の場合、フォントサイズを `42px`、行高（文字間隔）を `50px` に最適化。
    * 各列のX座標は、背表紙幅 `160px` に対して列数に応じた均等間隔（2列の場合は中央 `80px` から左右に `26.7px` ずらした `53.3px` と `106.7px`）で配置され、文字同士や金箔枠線との干渉・はみ出しを防ぐ。
    * 各列のY座標の開始位置は、それぞれの列の文字数に応じて独立して上下中央揃えになるよう計算され、ビジュアルバランスを維持する.
    * Comments本のタイトルは、表紙を `Comment Screen`、背表紙を `Comment Screen`（1列の縦積み表示、14文字のためフォントサイズ40px・行高46pxに自動縮小、空白は1スペースの余白として縦積みされる）として美しく構成。
