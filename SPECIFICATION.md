# SPECIFICATION

## 1. 概要
Three.js (WebGL) を用いた、アンティーク・クラシックな洋書風デザインの3D本オープニングアニメーションと、インタラクティブな本棚（Bookshelf）のフロントエンド実装。スマートフォン（縦画面）へのレスポンシブ対応と、文字の解像度向上を施したバージョン。

## 2. UX仕様（演出フロー）
### 2.1. オープニングシーン
* **初期状態**: 空間中央に閉じたアンティーク調の3Dの本（革張りのテクスチャ、高解像度の金箔押し文字）が配置されている。
* **アニメーション**:
  1. ページロード時、自動的に本がカメラに向かって滑らかに近づいてくる（Z軸移動）。
  2. 同時に、表紙が左側を軸に180度滑らかに開く。
  3. 開いた本の内側から眩い白色光が溢れ出す（ポイントライトの強度を急激に上げ、カメラを覆う）。
  4. 画面が完全に「真っ白（`#ffffff`）」にフェードアウトする。
  5. オープニングシーンのキャンバスを非表示にし、メインの本棚シーンへ移行。

### 2.2. メインシーン（本棚）
* **フェードイン**: ホワイトアウト状態から、重厚な木製の本棚がフェードインする（3D空間）。
* **背景デザイン**: 暖かみのあるロウソクや間接照明に照らされたような、ダークでアンティークな空間。
* **本棚構造**: 3Dで表現された木製の本棚に、4冊の洋書が背表紙を向けて並んでいる。
  * 各背表紙には項目名（`Profile`, `Works`, `Contact`, `Blog`）が金箔風のフォントで記載されている。
  * アスペクト比を監視し、スマートフォン等の縦長画面では本棚が画面内にすっぽり収まるように自動的にカメラを引き、配置を調整する。

### 2.3. インタラクション
* **ホバー（Hover / タッチ）**:
  * マウスカーソルを合わせる（またはスマホでタップしかける）と、その本が手前（Z軸）に引き出され、上に少し持ち上がる。
  * スマホでの操作性向上のため、タップ時は一度ホバー状態として引き出し、再タップでクリック遷移とするか、あるいはクリックアニメーションへ直接スムーズに移行する。
* **クリック（Click / タップ）**:
  * 本をクリックすると、手前中央に引き出される。
  * カメラの正面で本が回転し、表紙が少し開く。スマホ画面でも本が画面外にはみ出さないよう、引き出しスケールを画面アスペクト比に応じて動的にスケールダウンする。
  * その後、画面が暗転しハッシュ遷移（例: `#profile`）する。

## 3. 技術仕様
* **言語/ライブラリ**: HTML5, CSS3, JavaScript, Three.js (CDN), GSAP (CDN)
* **レスポンシブ・スマホ最適化**:
  * ウィンドウのアスペクト比（`width / height`）に基づき、カメラのZ座標（`camera.position.z`）を動的に計算する。
    * 横画面（PC）: `Z = 5.5`
    * 縦画面（スマホ）: アスペクト比に応じて `Z = 7.5` 〜 `8.5` 程度に後退させ、4冊の本が全て収まるように視野角（FOV）と位置を補正する。
  * 本の引き出しアニメーションの移動量をスマホ向けにスケールダウンし、画面外へのクリッピングを防ぐ。
  * CSSのメディアクエリ（`@media (max-width: 768px)`）を使用し、ヘッダー・フッターのフォントサイズやパディングをスマホに最適化する。
* **高解像度テキスト表現**:
  * `CanvasTexture` 用のキャンバスサイズを**4倍**（表紙: `1024x1024`、背表紙: `512x1024`）に拡張。
  * これに伴い、キャンバスに描画する飾り枠の線幅（`lineWidth`）やフォントサイズ（例: 表紙 `72px`、背表紙 `48px`）をスケールアップし、3D空間上でのジャギーやボケを排除し、非常にシャープでくっきりとした金箔文字を表現する。
* **コード構造**:
  * HTML、CSS、JavaScriptを単一のファイル（`index.html`）に内包する。
