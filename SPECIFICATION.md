# SPECIFICATION

## 1. 概要
Three.js (WebGL) を用いた、アンティーク・クラシックな洋書風デザインの3D本オープニングアニメーションと、インタラクティブな本棚（Bookshelf）のフロントエンド実装。スマートフォン（縦画面）への完全なレスポンシブ対応と、本を隙間なく並べることで実物の本棚のような密度感と画面収まりを両立したバージョン。

## 2. UX仕様（演出フロー）
### 2.1. オープニングシーン
* **初期状態**: 空間中央に閉じたアンティーク調の3Dの本（革張りのテクスチャ、高解像度の金箔押し文字）が配置されている。
* **アニメーション**:
  1. ページロード時、自動的に本がカメラに向かって滑らかに近づいてくる（Z軸移動）。
  2. 同時に、表紙が左側を軸に180度滑らかに開く。
  3. 開いた本の内側から眩い白色光が溢れ出す（ポイントライトの強度を急激に上げ、カメラを覆う）。
  4. 画面が完全に「真っ白（`#ffffff`）」にフェードアウトする。
  5. オープニングシーンのキャンバスを非表示にし、メインの本棚シーンへ移行。

### 2.2. メインシーン（本棚）
* **フェードイン**: ホワイトアウト状態から、重厚な木製の本棚がフェードインする（3D空間）。
* **背景デザイン**: 暖かみのある間接照明に照らされた、ダークでアンティークな空間。
* **本棚構造**: 3Dで表現されたコンパクトな木製の本棚に、4冊の洋書が背表紙を向けて**隙間なく並んで**格納されている。
  * 本の厚み（`0.5`）に対して並べる間隔を `0.52` に設定し、本同士の隙間をほぼゼロにすることで、本棚から引き出す感覚をリアルに再現する。
  * 本がギッシりと詰まったことで、全体の横幅がPC/スマホ問わずコンパクトになり、はみ出しを防ぐ。
  * 本棚全体の横幅スケールも、本が収まる最適なサイズ（幅 `5.0`）に縮小してバランスを整える。

### 2.3. インタラクション
* **ホバー（Hover / タッチ）**:
  * マウスカーソルを合わせる（またはスマホでタップする）と、隙間に挟まれていた本がスッと手前（Z軸）に引き出され、上に少し持ち上がる。隣の本との対比で引き出し演出が強調される。
* **クリック（Click / タップ）**:
  * 本をクリックすると、手前中央に引き出される。
  * カメラの正面で本が回転し、表紙が少し開く。スマホ画面でも本が画面外にはみ出さないよう、引き出しスケールを画面アスペクト比に応じて動的にスケールダウンする。
  * その後、画面が暗転しハッシュ遷移（例: `#profile`）する。

## 3. 技術仕様
* **言語/ライブラリ**: HTML5, CSS3, JavaScript, Three.js (CDN), GSAP (CDN)
* **レスポンシブ・スマホ最適化**:
  * 本の間隔（`spacing`）を `0.52` に設定。4冊の配置幅は `1.56` となり、従来の約半分以下にコンパクト化。
  * 本棚（木製棚板・背板・側板）の幅を `11.0` から `5.0` に縮小し、本の配置に完璧にフィットさせる。
  * ウィンドウのアスペクト比（`width / height`）に基づき、カメラのZ座標（`camera.position.z`）を動的に計算。
    * 横画面（PC）: `Z = 3.5`（本棚に近づけて迫力を出す）
    * 縦画面（スマホ）: `Z = 4.2 + (1.0 - aspect) * 2.0`（縦長になっても左右の本が絶対に見切れない距離に調整）
  * CSSのメディアクエリ（`@media (max-width: 768px)`）を使用し、ヘッダー・フッターのフォントサイズやパディングをスマホに最適化。
* **高解像度テキスト表現**:
  * `CanvasTexture` 用のキャンバスサイズ（表紙: `1024x1024`、背表紙: `512x1024`）と線幅・フォントサイズの最適化。
* **コード構造**:
  * HTML、CSS、JavaScriptを単一のファイル（`index.html`）に内包する。
